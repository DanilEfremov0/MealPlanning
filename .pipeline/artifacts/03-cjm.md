# 03 — Customer Journey Map
**Project:** MealPlanning
**Primary User Story:** US-02 — Создать план питания на неделю
**Persona:** Алина, 29 (основная) + Сергей, 36 (аллерген-сценарий)
**Date:** 2026-04-14

---

## Journey: Первое использование

### Стадия 1: Открытие приложения

| Элемент | Описание |
|---------|---------|
| Действие | Открывает ссылку, попадает на сайт |
| Точка контакта | Web browser / → redirect to /onboarding |
| Эмоция | Нейтральная — любопытство |
| Боль | «Ещё одно приложение, опять регистрация?» |
| Возможность | Никакой регистрации — сразу к делу |
| Метрика | Время загрузки < 2s; bounce rate |

Аналитика: не логируем (pre-onboarding)

---

### Стадия 2: Онбординг — настройка аллергенов

| Элемент | Описание |
|---------|---------|
| Действие | Видит вопрос про аллергии + чипы аллергенов |
| Точка контакта | /onboarding → AllergenPicker |
| Эмоция | Позитивная — «О, они думают о безопасности!» |
| Боль | Непонятно зачем это нужно |
| Возможность | Подзаголовок: «Мы исключим рецепты с этими ингредиентами» |
| Метрика | onboarding_allergens_set rate |

Аналитика: onboarding_started (mount) → onboarding_allergens_set (selection)

---

### Стадия 3: Генерация плана

| Элемент | Описание |
|---------|---------|
| Действие | Нажимает «Создать план питания» → попадает на Schedule |
| Точка контакта | CTA button → /schedule |
| Эмоция | Позитивное ожидание → Вау, это удобно! |
| Боль | Если генерация долгая (>2s) — тревога |
| Возможность | Skeleton loading; кнопка «Сгенерировать» prominent |
| Метрика | onboarding_completed; schedule_generated; time-to-first-schedule |

Аналитика: onboarding_completed → schedule_generated

---

### Стадия 4: Просмотр и корректировка плана

| Элемент | Описание |
|---------|---------|
| Действие | Листает дни, одно блюдо не нравится — нажимает, выбирает замену |
| Точка контакта | WeekGrid → RecipeCard → SwapModal |
| Эмоция | Контроль — «Я могу подстроить под себя» |
| Боль | Если альтернатив мало → «Не из чего выбрать» |
| Возможность | Показывать кол-во замен; copy «Перегенерировать всё» |
| Метрика | recipe_swapped rate; average swaps per session |

Аналитика: recipe_swapped (day, meal, from_recipe_id, to_recipe_id)

---

### Стадия 5: Список покупок

| Элемент | Описание |
|---------|---------|
| Действие | Переходит в Покупки, отмечает галочками в магазине |
| Точка контакта | Bottom nav → /shopping → checkbox taps |
| Эмоция | Облегчение — «Всё уже подсчитано» |
| Боль | Если без интернета — список недоступен |
| Возможность | Share button; offline через localStorage |
| Метрика | shopping_list_viewed; shopping_item_checked; shopping_list_shared |

Аналитика: shopping_list_viewed (mount) → shopping_item_checked → shopping_list_shared

---

### Стадия 6: Повторное использование

| Элемент | Описание |
|---------|---------|
| Действие | Следующая неделя — открывает сайт, нажимает «Перегенерировать» |
| Точка контакта | /schedule (onboarded=true, онбординг пропускается) |
| Эмоция | Привычка формируется |
| Боль | Старый план показывается — непонятно что ещё не создан новый |
| Возможность | Показывать дату генерации плана |
| Метрика | D7/D30 retention; repeat schedule_generated |

Аналитика: schedule_generated (повторный)

---

## CJM — Аллерген-сценарий (Сергей)

| Стадия | Особенность |
|--------|------------|
| Онбординг | Выбирает «Глютен» → подтверждает |
| Генерация | Все 21 слот — безглютеновые рецепты |
| Свап | В модале — только безглютеновые альтернативы |
| Покупки | «Поделиться» → отправляет жене через Web Share |

Критически важно: нужен индикатор «Глютен исключён» на экране расписания.
